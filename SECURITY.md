# セキュリティポリシー

## サポート対象バージョン

現在セキュリティアップデートを提供しているバージョンは以下の通りです。

| バージョン | サポート状況 |
| ----- | ------ |
| 1.x   | サポート中  |

最新のメジャーバージョンのみをサポートしています。

## 脆弱性の報告

セキュリティ脆弱性を発見した場合は、[GitHub Issues](https://github.com/semba-yui/atl-fetch/issues) で報告してください。

### 報告に含める情報

- 脆弱性の種類（例: インジェクション、認証バイパス）
- 影響を受けるファイル・機能
- 再現手順
- 影響範囲の評価
- 可能であれば、修正案

### 対応プロセス

1. **受領確認**: 報告を受けてから 3 営業日以内に確認の返信をします
2. **調査**: 報告された脆弱性を調査し、影響を評価します
3. **修正**: 脆弱性が確認された場合、修正パッチを作成します
4. **リリース**: 修正を含むバージョンをリリースします
5. **公開**: 修正リリース後、脆弱性の詳細を公開します

### 注意事項

- 脆弱性の悪用は行わないでください
- 報告前に脆弱性を公開しないでください
- 他のユーザーのデータにアクセスしないでください
